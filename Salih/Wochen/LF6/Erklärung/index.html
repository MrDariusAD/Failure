Einführung

In der Themenwoche "LF6 – Objekorientierte Modellierung und Programmierung II" ging es um den Einsatz von Klassen-, Objekt- und Sequenzdiagrammen. Da nicht alle abgegebenen Diagramme alle Kontrollstrukturen enthalten, gebe ich hier eine Erklärung zu den einzelnen Kontrollstrukturen ab.

Klassendiagramme

Abstrakte Klassen

Bei abstrakten Klassen wird der Name der Klasse kursiv geschrieben. Alternativ kann man unter den Namen die Bezeichnung "{abstract}" reinschreiben.

Assoziation

Eine Assoziation beschreibt eine Verbindung zweier Klassen, welche miteinander in Beziehung stehen. Im UML Klassendiagramm wird diese mit Hilfe eines einfachen Striches abgebildet.

Aggregation

Eine Aggregation beschreibt eine stärkere Beziehung zwischen zwei Objekten. Hierbei besitzt mindestens eine der beiden Klassen die jeweils andere. Im UML Klassendiagramm wird diese Beziehung mit einem Strich und einer nicht ausgefüllten Raute abgebildet. Die Raute befindet sich hierbei auf der Seite der besitzenden Klasse.

Komposition

Eine Komposition ist eine sehr besondere Beziehung. Bei einer Komposition ist eine Klasse von einer anderen abhängig. Man spricht hierbei davon, dass etwas ein Teil vom ganzen ist. Sollte also das ganze (z. B. ein Vogel) nicht mehr existieren, so existiert das Teil (z. B. ein Flügel) auch nicht mehr. Im UML Klassendiagramm bilden wir diese Beziehung mit einem Strich und einer ausgefüllten Raute auf der Seite des ganzen ab.

Vererbung

Bei einer Vererbung, erhält die sogenannte child class die Attribute und Methoden der parent class vererbt. Diese Beziehung wird im UML Klassendiagramm durch einen Strich mit einem Pfeil gekennzeichnet. Dabei zeigt der Pfeil auf die parent class.

Attribute

Attribute einer Klasse werden im Klassendiagramm wie folgt dargestellt:
<Zugriffsmodifier> <Variablenname> : <Datentyp>
Zugriffsmodifier können hierbei ein '-' für private, ein '+' für public oder ein '#' für protected sein.
Statische Variablen werden hierbei unterstrichen.

Methoden

Methoden sind den Attributen sehr ähnlich. Methoden einer Klasse werden im Klassendiagramm wie folgt dargestellt:
<Zugriffsmodifier> <Methodenname>(<Liste der Argumente>) : <Datentyp>
Zugriffsmodifier können hierbei ein '-' für private, ein '+' für public oder ein '#' für protected sein.
Statische Methoden werden auch hier unterstrichen.

Multiplizität

Die Multiplizität gibt an, wie viele Objekte der Klasse es innerhalb eines Programmes gibt. Multiplizitäten werden wie folgt dargestellt:
<Min>..<Max>
Für den Wert "unbegrenzt" setzen wir hier ein Stern('*') ein. Multiplizitäten befinden sich immer auf dem Strich der Beziehung. Hierbei ist zu beachten, dass die Werte an das gegenüber liegende # Ende kommen. Sprich sollte eine Playlist 0 bis unendlich Songs beinhalten können, so ist das "0..*" auf die Seite des Songs zu packen(siehe Bild).

Objektdiagramme

Objektdiagramme sind Klassendiagrammen sehr ähnlich. Sie unterscheiden sich allerdings darin, dass sie eine Instanz abbilden, und nicht das Programm generell. Dabei wird zu erst die Überschrift aus dem Klassendiagramm mit dem Objektnamen ergänzt (<Objektname> : <Klassenname>)

Beziehungen

Beziehungen werden hier genau so wie beim Klassendiagramm dargestellt.

Attribute

Attribute werden mit dem Inhalt des Attributs ergänzt. Dabei gilt folgende Darstellungsweise:
<Name des Attributs> = <Inhalt des Attributs>

Methoden

Methoden werden hier genau so wie beim Klassendiagramm dargestellt.

Sequenzdiagramme

Ein Sequenzdiagramm zeigt die Kommunikation zwischen Objekten innerhalb eines Softwaresystems. Das Sequenzdiagramm zeigt dabei die beteiligten Objekte, die Aktivitäten, die Nachrichten und die Lebenslinien der Objekte.

Objekte

Als Objekt zählt man in einem Sequenzdiagramm Akteure, sprich reelle Personen welche die Software bedienen, Datenbanken und Objekten innerhalb der Software. Akteure werden in Form eines Strichmännchens, Datenbanken als Kegel und andere Objekte als Rechteck dargestellt.

Lebenslinien

Lebenslinien werden durch gestrichelte Linien vom Objekt nach unten dargestellt. Das X zeigt hierbei das Ende der Lebenslinie eines Objektes. Lebenslinien zeigen die Lebenszeit einzelner Objekte.

Aktivitäten

Aktivitäten werden im Sequenzdiagramm mit Hilfe eines Balkens auf der Lebenslinie des ausführenden Objektes dargestellt. Aktivitäten zeigen an, wann die Objekte aktiv sind und Aktionen durchführen.

Nachrichten

Nachrichten zeigen die Interaktion bzw Kommunikation zwischen zwei Objekten. Dabei unterscheidet man zwischen synchronen und asynchronen Nachrichten. Synchrone Nachrichten werden mit einem ausgefülltem Pfeil und asynchrone mit einem leerem Pfeil dargestellt. Return Nachrichten werden mit einem gestricheltem Pfeil zurück dargestellt.
